Introduction

FASTDocs is a documentation layout for quickly creating a documentation from scratch. The advantages of FASTDocs are:

  • It has built-in indexing and search functionality.
  • Side-bar contents and site map are generated automatically.
  • Asset injection.
  • It is front-end only, no need to meddle with server-side code.
  • It is debranded and universal.

FASTDocs uses section HTML tag data-title, data-tags and data-desc attributes to index and separate and between contents. Everything can be done without writing a single line of JavaScript or server-side code.

Usage

First download the project (this documentation) from GitHub repository, then delete map.json and all the sections inside sections directory, this will give you clean workspace.

Later, write your own sections in pure HTML, put it in sections directory or any appropriate place you like.

Lastly, edit load.json file, add file path of newly created sections, and edit document title, title separator... to your liking.

Now you can start using your own documentation FASTDocs will handle the rest, optionally a new site map (map.json) can be exported for increased efficiency.

Creating Sections

Writing your own sections are easy as writing normal HTML code. Create a new pure HTML file without head and body tags and start writing your contents as you would normally do.

For content separation, FASTDocs utilizes section tag. These section tags alongside with any content inside of them then will be processed and mapped out for side-bar contents, search functionality and indexing.

There are 3 attributes needed to be defined inside section tags for processing and differentiation. These attributes are:

  • data-title: Title of your section, the title will be shown in side-bar and in search results.
  • data-tags: Tags of your section, tags are for searching functionality, add keywords related to your section for them to be searched easily. In order to add multiple tags, seperate them with a comma (,) inside the attribute.
  • data-desc: Description of your section, brief explanation to be shown in search results for your section.

Multiple sections can be inside one another to represent sub-sections.

A healthy section with a sub-section should look like this:

<section data-title="Title Of Your Section" data-tags="tag1, tag2, tag3" data-desc="Description for your section">
  <h1>Title</h1>
  <p>Lorem ipsum doler sit amet.</p>
  <section data-title="Sub Section Title" data-tags="tag4, tag5" data-desc="Description for sub-section">
    <h2>Sub-section</h2>
    <p>Lorem ipsum doler sit amet.</p>
  </section>
</section>

External Sections

Sections can be separated into different files and can be loaded from inside one another (instead of just load.json) with data-src attribute that contains the URL of the section to be retrieved. 

<section data-src="path/to/section.html"></section>

Site Map

Site map grants additional efficiency for indexing. Without a map, FASTDocs has to load every section first in order to index them, which is not ideal.

With a site map, FASTDocs only loads the site map at the initial process, and sections are loaded only when needed to show them.

map.json can be exported via download button on top navigation bar (don't forget to delete old map.json first).

Asset Injection, Theming, Scripting

FASTDocs already provides you with a sufficient and responsive user interface, however it is also possible to stylize your documentation with stylesheets or even execute scripts within sections.

Injecting styles and scripts are easy as you normally would do. If you want to add a style or script just add them to your sections using style, link, script tags. Your assets will be loaded when section is showed.

If you want to inject assets globally, insert your styles and scripts to assets.html file. Now your assets will be loaded at the initial process. FASTDocs will handle the assets automatically.

One thing to consider while injecting assets is that; FASTDocs ships with FontAwesome icons and custom stylesheet, so beware of clashing styles.

docs.onSectionLoad() function

Even though it is not necessary for you to write any scripts for FASTDocs to work, FASTDocs gives a one crucial function for your own scripts to be executed when a section is loaded.

First parameter for docs.onSectionLoad is a function which will be called. The second parameter is a boolean for whether the function will be executed once or every time. Example use case is given below:

docs.onSectionLoad(function (section) {
  console.log("Loaded: " + section.title + " once"); //This code piece will be executed only once
}, true);

docs.onSectionLoad(function (section) {
  console.log(section); //This code piece will be executed every time
});
